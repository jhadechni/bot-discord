# Identidad Visual

## Direccion general

Identidad: Discord nativo premium + branding Aquaris moderado.

Estetica:

- Limpia.
- Legible.
- Profesional pero amigable.
- No full business.
- No infantil.
- No sobrecargada.

Publico:

- Comunidad Minecraft casual/social.
- SMP privado.
- Usuarios jovenes y staff.

Prioridad maxima: legibilidad.

## Branding

El branding debe ser visible pero sutil:

- Footers coherentes.
- Colores consistentes.
- Tono claro y directo.
- Sin decoraciones pesadas.

## Elementos disponibles

Solo usar elementos viables en Discord.js:

- Embeds.
- Fields.
- Footer.
- Timestamp.
- Buttons.
- Select menus.
- Modals.
- Mensajes efimeros.
- DMs.
- Thumbnails cuando corresponda.

## Footers base

- `Aquaris Logs • Moderacion`
- `Aquaris • Moderacion`

- `Aquaris • Tienda`

## Regla clave

ID interno solo en logs y herramientas administrativas. Nunca en DM al usuario ni mensajes publicos.

## Principio modular

Cada modulo puede tener personalidad propia:

- Moderacion: seria, clara y auditable.
- Tienda: comercial y ordenada, sin parecer e-commerce corporativo.
- Reclutamiento: humano, calido y orientador.
- Recordatorios: practicos y directos.
- Sugerencias: comunitarias y transparentes.
- Niveles: sociales y motivadores, no excesivamente competitivos.
- Sistema: neutral y preciso.

Pero todos deben compartir estructura, colores, footers, tono limpio y reglas de exposicion de informacion.
